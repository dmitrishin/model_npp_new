## Финальный ТОП-10

Ничья на 9–10 позициях — обе идеи включены. Итоговый список содержит **10 идей** (на 10 позициях, 2 идеи делят 9–10 место).

### 1. Системный «шлюз первого бетона» — 59 баллов

**S02-R1-2-01** (Участник 2). Трёхисходная логика GO / CONDITIONAL GO / NO-GO на базе полного контура M-S-D-Q-U. Разделение предусловий на блокирующие (порог 1.0) и управляемые с остаточным риском. Профиль готовности, список блокеров, компенсирующие меры. Предотвращает вход в основной период с незрелым проектом.

### 2. Контур управления требованиями и Design Authority — 51 балл

**S02-R1-2-03** (Участник 2). Трёхклассовая типология требований (жёсткие / полуформальные / экспертные). Формальная оценка каскадного влияния изменений на сроки, стоимость, лицензирование и смежные системы. Эскалация на change board только критических изменений. Защита от «ползучих» переделок.

### 3. Дифференцированная волновая контрактация (buffered JIT) — 40 баллов

**S02-R1-2-05** (Участник 2). Три класса оборудования: А — ОДЦИ с обязательным буфером и жёстким заводским окном; B — среднецикловое с rolling-window; C — short-cycle с near-JIT. Привязка к пусковым комплексам и этапам ПНР. Оптимизация устойчивости критического пути, а не минимального склада.

### 4. Управление критическим путём ТМР — 28 баллов

**S02-R1-1-04** (Участник 1). Многоуровневая декомпозиция: системы → трубопроводные участки → сварные соединения. ML-прогнозирование длительностей на данных референтных проектов. Адаптивная детализация до швов только на критических участках. Интеграция с цифровыми журналами сварки.

### 5. Граф лицензирования и доказательной базы — 23 балла

**S02-R1-2-06** (Участник 2). Метрики полноты доказательств, покрытия требований, плотности замечаний. Встроен в общую модель проекта. Особенно ценен для экспортных проектов с двойным набором требований (национальный регулятор + МАГАТЭ).

### 6. «Заработанная готовность» — 19 баллов

**S02-R1-3-12** (Участник 3). Двухконтурная метрика: GateStatus (обязательные минимумы по каждому типу готовности) + ProgressScore (взвешенное геометрическое среднее физической, функциональной, документной и разрешительной готовности). Веса определяются профилем стадии. Замена процентов СМР на реальную близость к вводу.

### 7. Сквозная цепочка ИДЗ→закупка→ИДП→РД→СМР — 18 баллов

**S02-R1-1-09** (Участник 1). Полный жизненный цикл каждой позиции оборудования от проектной потребности до монтажа. Итерационный компонент Γ_iter с Nmax ≤ 3. Узлы принятия решений V_dec для перезапуска при замене поставщика. Ликвидация разрывов между проектированием, закупками и строительством.

### 8. Сквозной «двойник длительного цикла» для оборудования — 17 баллов

**S02-R1-3-05** (Участник 3). Связь проектных решений, ИДЗ/ИДП, контрактования, заводской готовности, логистики, помещений, монтажа, испытаний и ввода в состав системы. Показывает не только факт отставания, но и первопричину задержки.

### 9–10. Формализация условий «Первый бетон» — 16 баллов

**S02-R1-1-03** (Участник 1). Иерархический граф предусловий ПБ: лицензии, РД, котлован, СМБ, схема механизации. Метрики готовности τ_func с агрегацией α_min. Автоматическое выявление минимальных подмножеств незавершённых работ, блокирующих событие ПБ.

### 9–10. Управление ПНР через Stage-filtration — 16 баллов

**S02-R1-2-09** (Участник 2). Readiness-based контроль с различением c_phys / c_func. Формализованные условия переходов между подэтапами А-0...Г. Прогнозирование достижения функциональной готовности каждого пускового комплекса. Предотвращение ложной сдачи систем.
