## Раунд 2 — Критика

**S07-C-1-01**
ID идеи: S07-R1-4-01
Слабое место: Идея заявляет конкретные количественные результаты («сокращение риска срыва графика на 6–12 месяцев»), которые не имеют обоснования. Такие оценки невозможно дать до внедрения и апробации модели. Завышенные ожидания подрывают доверие к предложению и создают риск того, что при реальном внедрении результат будет воспринят как неудача, даже если модель принесёт значительную пользу.

**S07-C-1-02**
ID идеи: S07-R1-4-03
Слабое место: Применение «just-in-time» к поставкам оборудования АЭС противоречит специфике предметной области. Оборудование длительного цикла изготовления (ОДЦИ) имеет сроки производства 24–36 месяцев, а поставки подчиняются жёсткой логике шести волн контрактации. «Just-in-time» подразумевает минимизацию запасов и подстройку поставок под текущий момент потребности, тогда как в строительстве АЭС стратегически необходимы именно опережающие закупки и создание запасов. Идея также заявляет «снижение стоимости контрактации на 5–10%» без какого-либо обоснования.

**S07-C-1-03**
ID идеи: S07-R1-4-06
Слабое место: Идея предлагает «автоматическую генерацию и верификацию первоочередной рабочей документации». Рабочая документация АЭС — сложный инженерный продукт, разрабатываемый проектировщиками с учётом конкретных условий площадки, результатов изысканий и ИДП от поставщиков. Генерировать РД автоматически невозможно; модель может проверить полноту комплекта РД, прослеживаемость к ПД и соответствие классификациям, но не заменить проектировщика. Заявленное «сокращение времени подготовки РД на 3–5 месяцев» не обосновано.

**S07-C-1-04**
ID идеи: S07-R1-4-10
Слабое место: Идея описывает «полную оркестрацию U-уровнем AI-агентов для мониторинга и оптимизации всего жизненного цикла» — это максимально обобщённая формулировка, не содержащая конкретного механизма решения. Не описано, какие именно AI-агенты, какие решения они принимают, какие данные используют, как координируются с существующими процессами управления проектом. По сути, это заявление «применить ИИ ко всему» без содержательной проработки.

**S07-C-1-05**
ID идеи: S07-R1-6-01
Слабое место: Идея предлагает «развернуть структурную модель M на подготовительный период» — это описание первого шага внедрения любой из предложенных идей, а не самостоятельное предложение с конкретным механизмом решения. Все остальные идеи, использующие модель для подготовительного периода, неявно включают этот шаг. Идея не описывает, какую конкретную проблему и каким конкретным способом она решает сверх построения графа.

**S07-C-1-06**
ID идеи: S07-R1-6-02
Слабое место: Аналогично S07-R1-6-01 — идея описывает «использовать семантический слой S и классификации для создания терминологической базы». Это не самостоятельное применение модели, а необходимый инфраструктурный шаг для любого содержательного использования. Без указания конкретной задачи (управление изменениями, лицензирование, контрактация) идея остаётся описанием компонента модели, а не варианта её применения.

**S07-C-1-07**
ID идеи: S07-R1-5-11
Слабое место: Идея предлагает формировать «цифровой слепок» начального состояния оборудования для управления старением на горизонте 60–100 лет. Хотя задача управления старением реальна, применение модели сооружения к этой задаче выглядит натянутым: модель Дмитришина спроектирована для управления жизненным циклом сооружения (construction, commissioning), а не для эксплуатационного мониторинга деградации материалов. Для управления старением существуют специализированные методологии (IGALL, IAEA-TECDOC-1736). Кроме того, данные входного контроля и дефектоскопии фиксируются в паспортах оборудования и актах — привязка их к графовой структуре сооружения не даёт очевидной добавленной ценности по сравнению с существующими базами данных.

**S07-C-1-08**
ID идеи: S07-R1-5-12
Слабое место: Риск-информированная переклассификация оборудования — серьёзная регуляторная процедура, которая требует одобрения Ростехнадзора (или аналогичного зарубежного регулятора). Идея предлагает автоматизировать понижение класса безопасности через ИИ-агент и перестроение морфизма. На практике регулятор не примет результат автоматической переклассификации без детального обоснования по каждой позиции, и процесс принятия такого решения определяется не алгоритмом, а нормативно-правовой процедурой. Кроме того, ВАБ — это результат анализа на конкретный момент, а не постоянно доступный вычислительный ресурс.

**S07-C-1-09**
ID идеи: S07-R1-2-13
Слабое место: Идея предлагает моделировать геотехническую и гидрогеологическую неопределённость. Сама неопределённость носит физический, а не информационный характер: свойства грунтов и режим подземных вод выясняются в ходе инженерных изысканий и уточняются по мере проходки. Модель может отследить статус выполнения изысканий и связать результаты с проектными решениями, но она не может моделировать саму геотехническую неопределённость — для этого используются специализированные геомеханические модели (FEM, численные методы фильтрации). Существует риск того, что формальная графовая структура создаст иллюзию управления неопределённостью, которая на самом деле требует натурных данных.

**S07-C-1-10**
ID идеи: S07-R1-6-11
Слабое место: Автоматический расчёт EVM (SPI, CPI) через агрегацию атрибутов модели выглядит прямолинейно, но на практике основная сложность EVM — не в формулах, а в достоверности входных данных: фактический физический прогресс, фактические затраты, корректная привязка к WBS. Модель не решает проблему сбора достоверных данных с площадки — она лишь автоматизирует агрегацию данных, которые по-прежнему необходимо вводить вручную или получать из учётных систем. Если входные данные некачественны, автоматическая агрегация только быстрее распространит ошибку по всей иерархии.

**S07-C-1-11**
ID идеи: S07-R1-5-08
Слабое место: Оптимизация степени укрупнительной сборки через перебор пространства морфизмов Φ — математически красивая, но практически трудно реализуемая идея. Решение о стратегии монтажа (поэлементный vs. крупноблочный) определяется множеством факторов, которые сложно формализовать в виде морфизмов: фактическое наличие кранов на рынке аренды, состояние дорог, климатические окна для тяжёлых подъёмов, квалификация персонала подрядчика, конструктивные особенности конкретного оборудования. Перебор морфизмов может дать теоретически оптимальный вариант, который невозможно реализовать по практическим соображениям.

**S07-C-1-12**
ID идеи: S07-R1-3-02
Слабое место: Идея «обратного планирования от ввода блока через этапы ПНР до подготовительного периода» описывает стандартную методологию сетевого планирования (backward pass в CPM). Такой подход уже применяется при составлении типовых графиков сооружения — типовой график ОП-65 Росатома именно так и строится. Ценность модели Дмитришина в этом контексте не уникальна: любая система сетевого планирования (Primavera, MS Project) выполняет обратный проход. Идея не описывает, что именно графовая модель с семантикой, классификацией и корпусом добавляет сверх стандартного CPM backward pass.

**S07-C-1-13**
ID идеи: S07-R1-1-08
Слабое место: Межпроектное обучение через GNN требует наличия нескольких полностью заполненных экземпляров модели (графовая структура + атрибуты + временные ряды) по завершённым проектам. На данный момент таких экземпляров не существует — модель Дмитришина только опубликована. Первый экземпляр ещё нужно создать, заполнить данными, верифицировать. До появления обучающей выборки из хотя бы 3–5 проектов межпроектное обучение через GNN остаётся теоретической возможностью. Идея не описывает стратегию накопления данных и критерии достаточности обучающей выборки.

**S07-C-1-14**
ID идеи: S07-R1-6-09
Слабое место: Аналогично критике S07-R1-1-08 — применение GNN для выявления скрытых зависимостей предполагает наличие обучающей выборки из прошлых проектов в формате графов модели. Кроме того, в подготовительном периоде количество вершин и рёбер ограничено (десятки–сотни ключевых событий), что недостаточно для эффективной работы графовых нейронных сетей, требующих масштабных графов для обучения.

**S07-C-1-15**
ID идеи: S07-R1-4-08
Слабое место: Идея предлагает использовать модель для «динамического расчёта стоимости и финансирования подготовительного периода, включая cash-flow». Финансовое планирование и управление cash-flow — задача, которая решается специализированными финансовыми инструментами (ERP-системы, финансовые модели, казначейские системы). Модель Дмитришина не содержит механизмов для работы с финансовыми потоками, графиками авансирования, условиями оплаты по договорам. Применение графовой структуры жизненного цикла к расчёту cash-flow — это использование инструмента не по назначению.

Общий механизм: Формализация управления проектными изменениями (Change Impact Analysis) за счет отслеживания каскадных последствий изменения требований или документации через семантические связи на стоимость, сроки и смежные структуры в модели.

Группа: S07-R1-1-02, S07-R1-2-02, S07-R1-3-04, S07-R1-3-05, S07-R1-4-03, S07-R1-5-02, S07-R1-6-12
Общий механизм: Синхронизация графиков проектирования, поставок (ОДЦИ) и СМР в подготовительный период с использованием стохастических методов (PERT/GERT), предиктивной аналитики и планирования волн закупок.

Группа: S07-R1-1-06, S07-R1-2-04, S07-R1-2-09, S07-R1-3-11, S07-R1-4-02, S07-R1-5-06, S07-R1-6-02, S07-R1-6-10, S07-R1-6-14
Общий механизм: Использование семантических указателей и корпуса D для обеспечения сквозной доказательной базы (Traceability) и подтверждения статусов выполнения работ, особенно в части лицензирования и выполнения регуляторных требований.

Группа: S07-R1-1-08, S07-R1-2-12, S07-R1-3-10, S07-R1-6-06, S07-R1-6-09
Общий механизм: Межпроектный перенос знаний, извлечение уроков (Lessons Learned) и обучение ML-моделей на исторических данных портфеля завершенных проектов Росатома для предсказания рисков и узких мест на новых площадках.

Группа: S07-R1-1-10, S07-R1-2-03, S07-R1-3-08, S07-R1-4-05, S07-R1-5-04

## Раунд 2 — Критика

**S07-C-6-01**
**ID идеи:** S07-R1-1-01
**Слабое место:** Идея предлагает комплексную систему оценки готовности к первому бетону, но её реализация требует тотальной цифровизации всех процессов подготовительного периода и создания детальной модели с десятками тысяч элементов. Это колоссальный объём первичного наполнения и настройки правил агрегации, что может сделать внедрение на реальном проекте чрезмерно затратным и длительным, особенно на фоне сжатых сроков подготовительного периода. Существует риск «паралича анализа» вместо ускорения принятия решений.

**S07-C-6-02**
**ID идеи:** S07-R1-1-02
**Слабое место:** Использование GERT-подграфов для моделирования рисков импортозамещения требует точных вероятностных оценок для альтернативных путей (например, вероятность успешной замены поставщика). На практике, особенно в условиях быстро меняющейся геополитической обстановки, такие оценки могут быть крайне субъективными или недостоверными, что сведёт ценность вероятностного прогноза к нулю. Модель может создать иллюзию точного расчёта там, где царит высокая неопределённость.

**S07-C-6-03**
**ID идеи:** S07-R1-1-03
**Слабое место:** Формализация каждого изменения как трансформации, требующей проверки инвариантов, может создать бюрократический «бутылочное горлышко». В условиях реальной стройки, где требуется быстро реагировать на нештатные ситуации, процедура согласования через модель может стать слишком медленной. Строители будут искать обходные пути, чтобы не ждать формального подтверждения, и модель потеряет актуальность, а контроль — исчезнет.

**S07-C-6-04**
**ID идеи:** S07-R1-1-04
**Слабое место:** Идея сквозного графика двух периодов через множество Y*1 предполагает, что можно чётко определить переходные элементы. Однако в реальности многие работы подготовительного периода (например, корректировка РД по результатам поставок оборудования) могут влиять на основной период нелинейно и с большими задержками. Жёсткая фиксация границы может не отражать всю сложность взаимовлияний и привести к упрощённой, а значит, менее точной модели.

**S07-C-6-05**
**ID идеи:** S07-R1-1-12
**Слабое место:** Идея использования спектрального анализа графа (лапласианы, матрицы достижимости) для мониторинга стройки — это мощный, но очень сложный для восприятия инструмент. Руководители строительства, принимающие решения, вряд ли будут разбираться в «спектральных характеристиках». Чтобы идея работала, нужен чрезвычайно качественный и интерпретируемый интерфейс, который переводит математические аномалии на язык понятных инженерных рисков. Без этого она останется «чёрным ящиком» для практиков.

**S07-C-6-06**
**ID идеи:** S07-R1-2-10
**Слабое место:** Выделение интерфейсов как отдельного класса объектов в модели — полезно, но гиперрёбра «many-to-many» могут сделать граф чрезвычайно запутанным и трудночитаемым. Управление таким количеством связей, особенно если они не имеют чётких «владельцев» (Responsible), может привести к тому, что интерфейсы станут «ничьей землёй», а их статус будет постоянно неопределённым. Нужна очень строгая дисциплина назначения ответственных за каждое гиперребро.

**S07-C-6-07**
**ID идеи:** S07-R1-3-01
**Слабое место:** Сама по себе идея «единого шлюза» (как и многие другие, S07-R1-1-01, S07-R1-2-01) правильна, но она критически зависит от полноты и достоверности данных в корпусе D. Если в D загружены не все акты, или они «подписаны задним числом», или в них ошибки, то шлюз превратится в формальность. Проблема не в модели, а в культуре производства и документооборота, которую модель сама по себе не исправит.

**S07-C-6-08**
**ID идеи:** S07-R1-3-09
**Слабое место:** Идея «живой» модели рисков, привязанной к элементам графа, требует постоянного обновления вероятностей и влияний при каждом изменении. Это создаёт огромную нагрузку на риск-менеджеров и проектировщиков, которым придётся вручную переоценивать риски при каждом чихе. Без высокой степени автоматизации этой оценки (например, через ML, обученный на исторических данных) модель рисков быстро устареет и перестанет быть «живой».

**S07-C-6-09**
**ID идеи:** S07-R1-4-03
**Слабое место:** Идея предиктивного управления поставками через ML-агентов обещает снижение стоимости на 5-10%. Однако ML-модели требуют большого объёма качественных исторических данных по срывам поставок, задержкам ИТТ, конкурсным процедурам. Такие данные в Росатоме, скорее всего, разрозненны, неструктурированы и не привязаны к единой классификации. Очистка и подготовка таких данных — это отдельный огромный проект, без которого ML не запустится, а обещанный эффект не будет достигнут.

**S07-C-6-10**
**ID идеи:** S07-R1-5-05
**Слабое место:** Идея управления передачей систем через итеративные блоки Γ_iter с жёстким правилом STRICT (система готова только при готовности всех дочерних элементов) — это идеально с точки зрения качества, но может быть нереалистично на стройке. Часто ПНР по системе начинается, когда часть оборудования уже работает, а часть ещё доделывается, но это не блокирует пусконаладку в целом. Слишком жёсткое правило может создать искусственные блокеры и затянуть график, не повысив безопасность.

**S07-C-6-11**
**ID идеи:** S07-R1-5-08
**Слабое место:** Идея выбора оптимальной степени укрупнения через перебор морфизмов Φ требует, чтобы все возможные варианты монтажа (поэлементный, блочный) были заранее спроектированы и просчитаны. На практике это колоссальный объём проектных работ, который часто невозможен в сжатые сроки подготовительного периода. Кроме того, оптимизация по критерию «минимум сроков» без учёта стоимости сверхтяжёлой крановой техники может привести к экономически невыгодному решению.

**S07-C-6-12**
**ID идеи:** S07-R1-5-10
**Слабое место:** Идея сглаживания пиковой численности персонала путём сдвига некритических задач — это стандартная техника ресурсного планирования. Однако в атомном строительстве технологическая последовательность работ (например, бетонирование -> выдержка бетона -> гидроизоляция) часто не оставляет свободы для манёвра. Сдвиг задач в пределах их резерва может быть невозможен из-за технологических требований или погодных условий. Модель может предложить теоретически оптимальный, но практически невыполнимый график.

**S07-C-6-13**
**ID идеи:** S07-R1-6-01
**Слабое место:** Моя собственная идея. Хотя она предлагает создать «цифровой двойник графика», она не уточняет, как именно этот двойник будет синхронизироваться с реальностью. Если данные о прогрессе будут вноситься вручную и с задержкой, то ценность модели для оперативного управления будет низкой. Не хватает проработки механизма сбора фактических данных (например, интеграция с ИСУП, фотофиксация, датчики на технике).

**S07-C-6-14**
**ID идеи:** S07-R1-6-06
**Слабое место:** Моя идея о предиктивной аналитике. Критически зависит от качества «исторических данных». Если данные по прошлым проектам хранятся в разных форматах, с разной детализацией и классификацией, то обучение на них будет бесполезным. Идея не учитывает огромный объём предварительной работы по гармонизации данных со всех предыдущих строек, который может занять годы.
